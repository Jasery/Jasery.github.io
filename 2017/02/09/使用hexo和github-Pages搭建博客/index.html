使用hexo和github Pages搭建博客系统

撰写于 2017-02-09 修改于 2017-02-19 分类 教程 标签 blog

此文章适合知道github,知道命令行，想搭建博客的朋友

这次算是把博客搭好了，第一篇文章(helloworld除外)就用来聊聊搭建博客吧。

名词解释

在我刚开始逛github的时候，发现很多人的博客都长一样，而且经常出现一个名词“hexo”。如果可以事先对要学习的东西有个概念，后面的进行会方便很多

github Pages

  • 首先是github,简单说就是一个可以保存和分享代码的平台，是目前最大的开源社区，有许许多多优秀的开源项目。github使用git作为版本控制，如果需要了解git的话请移步git book。
  • github pages，是给github提供的一个免费的资源服务，你在github上按照指定的方式创建一个仓库，就可以通过浏览器在指定的域名下访问到你上传的资源。更多请参考https://pages.github.com/。

hexo

  • Markdown,用hexo搭建的博客，文章需要用markdown写。Markdown是一种可以让你的文本按照指定的格式去显示的轻量级标记语言，相对比较简单。如果你有HTML基础，markdown的难度几乎可以忽略。需要了解更多请移步认识与入门Markdown

    补充：hexo不仅仅支持Markdown,还有EJS和Stylus等,但目前主流的是Markdown

  • hexo,读作hack so,没人在我面前读过，我们暂且就这样读吧。hexo，今天的主角，可以把你用markdown写的文章，按照指定的主题模板生成静态页面。然后我们把这些页面放到github pages上，就可以让别人浏览到了。

准备工作

安装

安装Node.js和Git，如果不知道这两个东西可以先不用管，装就是了。windows下下载Node.js和Git(本教程默认都是在windows下，其他平台类似)并安装。Git装好后，点击鼠标右键可以看到有个Git bush here,可在当前目录下打开一个类Linux命令窗口。后面的命令行都可以这样打开去执行。打开命令行后，可以通过命令行node -v查看node版本检查Node.js是否安装好。
装好Node.js和Git后，就可以装hexo了，只需要执行命令npm install -g hexo-cli，同样可以通过查看版本命令hexo v检查是否安装成功。

hexo命令

基本命令

hexo的命令不多，常用的就更少了，这里把常用命令做一个说明。

命令 说明
hexo init myDir 在当前目录下初始化一个hexo项目，保存在myDir文件夹
hexo generate 生成静态网页文件
hexo server 开启预览，默认地址是http://localhost:4000
hexo deploy 部署网站，比如我们用github pages的话这个命令可以帮我们把页面提交到github上
hexo new myArticle 新增一篇标题为myArticle的文章

hexo命令流程

注意：
hexo init初始化后，一定要进入工程文件夹执行npm install命令，这个命令会把一些依赖的模块全部安装，保存在node_modules文件夹里面，没有这个安装命令则后面的都执行不了

配置博客

初始化完了一个hexo之后，其实就已经有了套默认的页面（默认主题）了，而博客的很多内容都是通过写配置去设置。我们可以看到其他人的博客都是做的很好的页面，换做自己肯定做不出这么好的页面。这些都是归功与主题。本博客用的主题是random，后面将会以这个主题为例，讲讲如何配置主题。更多主题请移步hexo主题，佳丽三千，总能找到一个适合你的。

基本配置

初始化好一个hexo工程之后，在工程文件夹里面多了很多文件夹和文件，下面简单描述各文件或文件夹的作用

文件/文件夹 说明
public 里面是hexo根据我们的文章和主题生成的页面文件
scaffolds 用于保存新建文章的模板
source 用于保存我们的资源，比如文章，图片等
themes 保存hexo主题
_config.yml hexo的配置文件

hexo的配置都是写到_config.yml文件里，包括后面要将的主题的配置，只是主题的配置在对应主题文件夹里。hexo的配置有的很多，大部分都是保留默认的就可以了,以下是部分配置说明

配置项 说明
title 标题
subtitle 子标题
description 描述，用于方便被别人搜索到
author 作者
languange 语言，中文用zh-CN
theme 使用的主题，需要先把主题下载到themes文件夹
deploy 部署的配置

说明：

  • 其他配置可以到hexo官网查看
  • 配置的值跟前面的冒号需要有一个空格隔开，不然会报错
  • 部署使用github的话需要安装hexo-deployer-git插件，就是执行npm install hexo-deployer-git --save命令

主题配置

我们在hexo工程根目录配置好主题（注意，这里配置值一定要跟保存在themes文件夹下的主题的文件夹名称一致），就可以看到精美的页面了。一般hexo主题在项目的github都会有说明。

部署

github pages 使用方法很简单，只需要创建一个以你的用户名加上.github.io命名的仓库就可以了，比如我的就是jasery.github.io仓库，然后往这个仓库提交hexo生成的文件，通过网址https://github pages仓库名就可以访问了，但是每次hexo生成完了之后，都要提交到仓库的话，显得也太麻烦了。事实上，只需要hexo deploy命令就已经可以完成了所有操作，当然需要提前配置好。

结束

到这里，一个博客系统就基本搭建好了，以后只需要添加文章，生成，预览，部署，简单的几条命令就可以完成更新。如有需要帮助可以通过About me联系我。
